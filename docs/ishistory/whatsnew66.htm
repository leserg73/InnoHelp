Модернизация графики в инсталляторе/деинсталляторе

Программа установки и удаления теперь поддерживает тёмную тему: принудительно или автоматически.

Все эти параметры являются дополнительными, имеют обратную совместимость и НЕ увеличивают размер инсталлятора, если не используются.

  • В секции [Setup] директива WizardStyle дополнена новыми параметрами:
    • Добавлены режимы внешнего вида: light (по умолчанию, стандартная светлая тема, что и в предыдущих версиях), dark (принудительная тёмная тема, новый внешний вид) и dynamic (автоматическая установка темы в зависимости от системных настроек темы Windows).
    • Например, чтобы использовать современный стиль с автоматическим выбором между светлой и тёмной темами в зависимости от настроек Windows, следует указать директиву со следующим значением: 
      WizardStyle=modern/dynamic
    • Тёмная тема автоматически заменяет значки, изображения мастера и настраивает цвета, сохраняя при этом единообразный интерфейс. Также по-прежнему доступна настройка главного значка, изображений мастера и цветов. Включает поддержку стилизации диалоговых окон задач.
      Скриншот окна инсталлятора с тёмным стилем при DPI 175%.
      Скриншот диалога задач с тёмным стилем при DPI 125%.
    • Для получения дополнительной информации смотрите обновленный раздел справки с описанием директивы WizardStyle.
  • Добавлена новые директивы DynamicDark в секции [Setup] для указания отдельных пользовательских изображений и цветов, которые специально будут использоваться в режиме dynamic при переключении на тёмную тему.
    • Например, укажите два пользовательских цвета: один для светлой темы, а другой для тёмной: 
      WizardStyle=modern/dynamic
WizardImageBackColor=$ffffff
WizardImageBackColorDynamicDark=$2b2b2b
      Примечание: принудительная установка тёмной темы (classic/dark или modern/dark) не использует настройки DynamicDark; она использует стандартные настройки. Например: 
      WizardStyle=modern/dark
WizardImageBackColor=$2b2b2b
  • Стиль по умолчанию можно заменить на другой файл VCL-стиля с помощью новой директивы WizardStyleFile и директивы DynamicDark.
    • Например, чтобы использовать другие стили для светлой и тёмной тем, укажите соответствующие файлы стилей: 
      WizardStyle=modern/dynamic
WizardStyleFile=Windows11_Polar_Light.vsf
WizardStyleFileDynamicDark=Windows11_Polar_Dark.vsf
      Скриншот инсталлятора со стилем Polar Light при DPI 175%.
      Скриншот инсталлятора со стилем Polar Dark при DPI 175%.
      Скриншот диалога задач со стилем Zircon при DPI 125%.
    • Если принудительно активирована светлая (light) тема или dynamic тёмная (dynamic с Windows в светлом режиме), стилем по умолчанию является встроенная системная светлая тема. В противном случае – встроенная пользовательская тёмная тема.
    • Если активен пользовательский стиль (включая встроенную пользовательскую тёмную тему), все текстовые метки, за исключением метки, связанной с сообщением BeveledLabel, имеют прозрачный фон.
  • Программирование на языке Паскаль: добавлены функции IsWinDark и IsDarkInstallMode.
Другие изменения
  • Программирование на языке Паскаль: добавлена поддержка кнопок командных ссылок (стиль кнопок, используемый в диалоговых окнах задач). Смотрите пример в сценарии CodeClasses.iss.
