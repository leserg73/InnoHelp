[Setup]: WizardSmallImageFile

Значение по умолчанию:
(пустое)

Описание

Эта директива определяет малое изображение, которое отображается с левой стороны на страницах мастера установки/удаления. Принимает один или несколько файлов изображений в формате BMP, также поддерживает маску файловых имён. При сборке инсталлятора файлы изображений должны находиться в исходном каталоге сценария, в противном случае к изображениям необходимо указывать полные пути. При использовании приставки "compiler:", компилятор будет искать эти файлы в собственном каталоге.

Поддерживаются изображения в формате .bmp и .png, включая прозрачность в этих изображениях. Используйте директиву WizardImageBackColor для настройки цвета фона. Возможно использование .bmp файлов с прозрачностью (дополнительную информацию смотрите в разделе WizardImageAlphaFormat).

Размер области, в которой выводится изображение, зависит от системных настроек DPI и используемых параметров шрифта. Во всех случаях эта область квадратная, поэтому и указанное изображение должно быть квадратным.

Можно использовать изображение любого размера. Если изображения слишком малы или слишком велики, чтобы поместиться в область вывода изображения, то они будут растянуты или сжаты. Рекомендуется указывать изображение размером более 58x58 пикселей. Это позволяет избежать размытости и нечеткости изображения на системах с высоким уровнем DPI. По умолчанию размер встроенного изображения составляет 147x147 пикселей.

Если одно изображение большого размера при разных настройках DPI не даёт удовлетворительного результата, можно указать несколько файлов, разделив их запятыми. Программа установки автоматически выберет из них то, которое лучше всего соответствует размеру области изображения. При условии использования настроек шрифта по умолчанию размер области изображения с различными настройками DPI будет составлять (в пикселях):

100% 58x58
125% 77x77
150% 97x97
175% 116x116
200% 124x124
225% 143x143
250% 159x159

(Можно заметить, что размер в 200% не соответствует удвоенному размер в 100%. Это обусловлено тем, что коэффициент масштабирования основан на размере шрифта, который используется при настройке DPI, а не на самом DPI).

Примечание об обратной совместимости: если указано одно изображение размером менее 58x58 пикселей, оно не будет растягиваться, чтобы заполнить всю область вывода изображения; вместо этого оно будет выровнено по центру.

Если эта директива не указана или имеет пустое значение, то будет использоваться одном встроенное изображение мастера с размерами 147x147 пикселей, которое будет сжато или растянуто до необходимых размеров.

Если используются настройки по умолчанию, установленные до версии Inno Setup 6.6.0, размер области изображения при различных настройках DPI будет следующим (в пикселях):

100% 58x58
125% 71x71
150% 85x85
175% 103x103
200% 112x112
225% 129x129
250% 147x147

Чтобы вновь использовать старое малое изображение мастера установки, укажите директиву со значением compiler:WizClassicSmallImage.bmp.

Пример:
WizardSmallImageFile=mysmallimage.bmp,mysmallimage2.bmp
