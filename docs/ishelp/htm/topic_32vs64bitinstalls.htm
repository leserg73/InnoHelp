64-разрядный режим установки

Установка может выполняться в одном из двух режимов: 32-разрядном или 64-разрядном. По умолчанию используется 32-разрядный режим установки. Для включения 64-разрядного режима установки доступно два варианта:

  • Установите для директивы SetupArchitecture значение x64 (рекомендуется). Это создаст 64-разрядный (x64) установщик и дополнительно активирует 64-разрядный режим установки (по умолчанию).
  • В качестве альтернативы воспользуйтесь директивой, ArchitecturesInstallIn64BitMode для включения 64-разрядного режима установки в 32-разрядном установщике.

Также можно использовать 32-разрядный режим установки в 64-разрядном установщике (см. раздел "Inno Setup 64-бит".

Чем же различаются эти два режима работы? Главное различие состоит в значениях констант, которые используются по умолчанию.

32-разрядный режим установки:

  • Все записи по умолчанию являются 32-разрядными. Обратитесь к разделу "Разрядность записи" для получения дополнительной информации.
  • Константа {commonpf} эквивалентна константе {commonpf32}.
  • Константа {commoncf} эквивалентна константе {commoncf32}.
  • Константа {reg:...} по умолчанию выполняет чтение из 32-разрядного представления реестра.
  • Функции Reg* секции [Code] по умолчанию получают доступ к 32-разрядному представлению реестра.
  • Ключи деинсталляции создаются в 32-разрядном представлении реестра.

64-разрядный режим установки:

  • Все записи по умолчанию являются 64-разрядными. Обратитесь к разделу "Разрядность записи" для получения дополнительной информации.
  • Константа {commonpf} эквивалентна константе {commonpf64}.
  • Константе {commoncf} эквивалентна константе {commoncf64}.
  • Константа {reg:...} по умолчанию выполняет чтение из 64-разрядного представления реестра.
  • Функции Reg* секции [Code] по умолчанию получают доступ к 64-разрядному представлению реестра.
  • Ключи деинсталляции создаются в 64-разрядном представлении реестра.
